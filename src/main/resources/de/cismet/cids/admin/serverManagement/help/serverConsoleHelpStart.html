ServerConsole starten

Die ServerConsole wird wie folgt aufgerufen:

ServerConsole -t serverType -n serverClassName [-l log4jConfig] [-p port] [-s serverManagementRoot] [-c miniatureServerConfig] [-a cidsServerArgs]

-t serverType Bezeichnung des Servertyps (z.B. "cids Registry")
-n serverClassName Vollständiger Klassenname des Servers.
-l log4jConfig Log4j-Konfigurationsdatei. Wird keine angegeben, wird der Log4j BasicConfigurator benutzt.
-p port Port, auf dem das Management-Webinterface laufen soll. Wird kein Port angegeben, wird das Webinterface nicht gestartet.
-s serverManagementRoot Arbeitsverzeichnis des cids Servermanagements. Wenn keins angegeben wird, wird das Verzeichnis, aus dem die ServerConsole gestartet wurde, zum Arbeitsverzeichnis.
-c miniatureServerConfig Konfigurationsdatei für den MiniServer. Wenn keine Konfigurationsdatei für den MiniServer angegeben wird, wird die Standard-Konfigurationsdatei aus dem Classpath genommen.
-a cidsServerArgs (Ein oder mehrere) Parameter für den jeweiligen cids Server. Ist unter den Parametern eine Konfigurationsdatei für den cids Server, muss dieser Parameter als erster hinter dem Schalter -a stehen.
Die oben aufgeführten Konfigurationsdateien können absolut und relativ angegeben werden. Bei einer relativen Angabe wird das Arbeitsverzeichnis des cids Servermanagements als Basis genutzt. Wenn der Schalter -a mit Parametern angegeben wird, muss er der letzte sein.
